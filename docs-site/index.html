Quick Start

Get from zero to a live policy engine in under 5 minutes.

1. Create an account

curl
curl -X POST https://api.agentguard.tech/api/v1/signup \
  -H "Content-Type: application/json" \
  -d '{"name": "Acme Corp", "email": "security@acme.com"}'

Response includes your apiKey (format: ag_live_ + 32 hex chars). Store it securely — it's only shown once.

2. Install the SDK

TypeScript
Python
npm install @the-bot-club/agentguard
pip install agentguard-tech

3. Evaluate an action

TypeScript
Python
cURL
import { AgentGuard } from '@the-bot-club/agentguard';

const guard = new AgentGuard({
  apiKey: 'ag_live_your_key_here',
  baseUrl: 'https://api.agentguard.tech'
});

const decision = await guard.evaluate({
  tool: 'http_post',
  params: { destination: 'https://api.stripe.com/v1/charges', amount: 4200 }
});

if (decision.result === 'allow') {
  // Safe to execute
  await executeAction();
} else {
  console.log(`Blocked: ${decision.reason}`);
}
from agentguard import AgentGuard

curl -X POST https://api.agentguard.tech/api/v1/evaluate \

That's it. Every action your agent takes now goes through AgentGuard's policy engine before execution.

Authentication

All authenticated endpoints require the X-API-Key header.

X-API-Key: ag_live_your_key_here

AgentGuard uses two types of API keys:

Key FormatTypeDescription
ag_live_*tenantTenant-level key. Full access to all endpoints for your account.
ag_agent_*agentAgent-scoped key. Restricted to evaluate + audit. Policies scoped to this agent.
No API key? Requests without a key run in demo mode — fully functional but data isn't persisted. Great for testing.

SDK Installation

TypeScript / Node.js

npm install @the-bot-club/agentguard
import { AgentGuard } from '@the-bot-club/agentguard';

const guard = new AgentGuard({
  apiKey: process.env.AGENTGUARD_API_KEY,
  baseUrl: 'https://api.agentguard.tech'  // optional, this is the default
});

// Core methods
await guard.evaluate({ tool, params });
await guard.getAudit({ limit, offset });
await guard.verifyAuditChain();
await guard.getUsage();
await guard.setKillSwitch(true);

// Agent management
await guard.createAgent({ name, policyScope });
await guard.listAgents();
await guard.deleteAgent(agentId);

// Webhooks
await guard.createWebhook({ url, events, secret });
await guard.listWebhooks();
await guard.deleteWebhook(webhookId);

// Compliance templates
await guard.listTemplates();
await guard.getTemplate(name);
await guard.applyTemplate(name);

// Rate limiting
await guard.createRateLimit({ windowSeconds, maxRequests });
await guard.listRateLimits();

// Cost attribution
await guard.trackCost({ agentId, tool, estimatedCost, currency, metadata });
await guard.getCostSummary({ period, groupBy });

// Dashboard
await guard.getDashboardStats();
await guard.getDashboardFeed({ limit });
await guard.getDashboardAgents();

Evaluate

The core endpoint. Submit an agent action for policy evaluation before execution.

POST /api/v1/evaluate Auth Optional

Request Body

FieldTypeDescription
toolstringrequiredAction being attempted (e.g. file_read, http_post, shell_exec)
paramsobjectoptionalParameters of the action (e.g. {"path": "/etc/passwd"})
contextobjectoptionalAdditional context (user, session, environment)

Response — 200

{
  "result": "block",
  "reason": "Blocked by rule \"block-privilege-escalation\"",
  "riskScore": 85,
  "rule": "block-privilege-escalation",
  "durationMs": 0.06,
  "sessionId": "demo-abc123",
  "hashChain": "sha256:a1b2c3..."
}

Result Values

ValueMeaning
allowAction is safe to execute
blockAction violates a policy rule — do not execute
hitl_requiredHuman-in-the-loop approval needed before execution
Without an API key, requests run in demo mode — functional evaluation against default rules, but decisions aren't persisted.

Audit Trail

Every evaluation decision is recorded in a SHA-256 hash-chained audit log. Each entry includes the hash of the previous entry, making the chain cryptographically tamper-evident.

GET /api/v1/audit Auth Required

Query Parameters

FieldTypeDefaultDescription
limitnumber50Max events to return (1–500)
offsetnumber0Pagination offset

Response — 200

{
  "events": [
    {
      "id": "evt_...",
      "tool": "shell_exec",
      "result": "block",
      "rule": "block-privilege-escalation",
      "riskScore": 95,
      "durationMs": 0.04,
      "hashChain": "sha256:...",
      "agentId": "agent-1",
      "timestamp": "2026-03-02T02:15:00.000Z"
    }
  ],
  "count": 1,
  "total": 42
}

Verify Hash Chain

GET /api/v1/audit/verify Auth Required

Cryptographically verify the integrity of your audit trail. Returns valid: true if no entries have been tampered with.

Webhook Alerts

Get real-time notifications when actions are blocked, the kill switch activates, or human review is required. Webhooks include HMAC-SHA256 signatures for verification.

POST /api/v1/webhooks Auth Required NEW

Request Body

FieldTypeDescription
urlstringrequiredHTTPS endpoint to receive events
eventsstring[]requiredEvent types: block, hitl, killswitch
secretstringoptionalHMAC secret for payload verification

Webhook Payload

{
  "event_type": "block",
  "tenant_id": "a80dd96c-...",
  "agent_id": "booking-agent",
  "data": {
    "decision": "block",
    "rule": "block-external-api",
    "tool": "http_post",
    "riskScore": 75
  },
  "timestamp": "2026-03-02T02:15:00.000Z"
}
HMAC verification: If you set a secret, every payload includes an X-AgentGuard-Signature header with sha256=HMAC(payload, secret).

List all webhooks for your tenant.

Delete a webhook.

Policy Engine

The policy engine evaluates 7 rule types against every action, in priority order:

Rule TypeDescriptionExample
resource_accessBlock or allow access to specific tools/resourcesBlock shell_exec, allow file_read
rate_limitLimit action frequency per time windowMax 10 API calls per minute
scope_restrictionRestrict parameter valuesOnly allow http_post to whitelisted domains
hitl_gateRequire human approval before executionFlag any action over $1,000
time_windowAllow/block actions based on timeNo deployments outside business hours
environmentEnvironment-specific rulesBlock destructive actions in production
data_classificationProtect sensitive data patternsBlock access to PII-classified resources

Rules are evaluated in priority order (lowest number first). The first matching rule determines the result. If no rule matches, the default action applies (configurable, defaults to allow).

Error Handling

All errors return JSON with an error field:

{ "error": "Invalid or missing API key" }
StatusMeaning
400Bad request — missing or invalid parameters
401Unauthorized — invalid or missing API key
404Not found — endpoint or resource doesn't exist
429Rate limit exceeded — includes resetAt timestamp
500Internal server error
