E.3. Innovation Description

Describe your project and how it is innovative (max 2000 chars):

AgentGuard is a runtime security platform for AI agents — the first product to combine deployment-time scanning with sub-millisecond runtime policy enforcement and tamper-evident audit trails in a single platform. AI agents built with LangChain, CrewAI, AutoGen, and OpenAI frameworks can execute SQL queries, run shell commands, call APIs, and modify files. Currently, no standardised security controls govern these actions. Prompt injection attacks can trick agents into executing destructive operations. The EU AI Act (enforcement August 2026) will require runtime monitoring and audit trails for high-risk AI systems. Our innovation addresses this gap with three enforcement points: 1. Runtime Policy Engine: A sub-millisecond evaluation API that intercepts every tool call an AI agent makes. Each call is evaluated against configurable policies (block/allow/monitor/human-approval) in under 1ms — fast enough to sit in the agent's execution loop without latency impact. 2. Tamper-Evident Audit Trail: Every evaluation decision is logged with a SHA-256 hash chain, creating a cryptographically verifiable record. This enables compliance evidence for the EU AI Act, SOC 2, APRA CPS 234, and the Australian AI Ethics Framework. 3. Framework Integrations: Drop-in SDKs for 7 major agent frameworks (TypeScript and Python), requiring 3 lines of code to add security to an existing agent. No existing product combines these capabilities. Container security vendors (Snyk, Wiz) scan images but don't inspect agent behaviour. AI governance platforms (Credo AI) focus on model bias, not runtime execution. AgentGuard is first to apply DevSecOps security paradigms to the AI agent lifecycle. Current status: v0.9.0 live, 60+ API endpoints, 193+ automated tests, published SDKs on npm and PyPI, production deployment on Azure.

NRF alignment (max 2000 chars):

AgentGuard aligns with the NRF priority area of "Enabling Capabilities" as foundational security infrastructure for Australia's AI industry. ENABLING CAPABILITIES: As Australian enterprises across all NRF sectors (resources, agriculture, medical science, defence, renewables) adopt AI agents, they need security controls to deploy these agents safely and in compliance with regulations. AgentGuard is an enabling technology — it doesn't compete with AI agent frameworks but makes them safe for production use. The Australian AI Ethics Framework recommends audit trails and human oversight for AI decision-making. AgentGuard implements these requirements at the technical level, enabling Australian businesses to adopt AI agents responsibly. APRA CPS 234 requires regulated financial entities to maintain information security controls proportionate to threats. AI agents with database and API access represent a new threat vector that existing security tools don't address. AgentGuard fills this gap for Australian financial institutions (CBA, ANZ, Macquarie, AMP) deploying AI agents. DEFENCE CAPABILITY: The Australian Department of Defence AI Strategy emphasises responsible AI deployment, particularly for autonomous systems. AgentGuard's policy enforcement and audit trails directly support Defence AI governance requirements for human-on-the-loop supervision. Our kill switch capability enables instant halt of all agent operations — critical for defence and national security applications. The EU AI Act (August 2026) creates global compliance demand. Australian companies selling to EU customers need AI audit trails. AgentGuard provides this capability, positioning Australian software as a global enabler of AI compliance. Building AgentGuard in Australia creates sovereign capability in AI security — reducing dependence on US vendors and creating exportable technology for the global market.
